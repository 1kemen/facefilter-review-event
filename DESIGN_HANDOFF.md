## 현재 고객 화면 핵심 플로우

1. 참여 정보 입력
2. 네이버 리뷰 작성하기
3. 리뷰 작성 완료 버튼 클릭
4. 물방울 선택
5. 선택한 물방울로 상품뽑기 확정
6. 카카오톡 채널 혜택 선택
7. 직원 확인 화면

이 플로우를 깨지 않도록 디자인만 정리해야 합니다.

## 디자인 작업 주의사항

- 모바일 화면을 우선으로 봅니다.
- 입력칸은 가급적 세로 배치가 안정적입니다.
- 버튼 문구가 두 줄로 어색하게 깨지지 않게 합니다.
- 고객 화면은 최대한 짧고 명확해야 합니다.
- 관리자/직원 화면은 정보량이 많아도 액션 버튼이 잘리지 않게 합니다.
- `hidden`, `id`, `data-*` 속성은 기능과 연결되어 있으므로 임의로 삭제하지 않습니다.
- 버튼 `id`는 바꾸지 않습니다.
- 모달, 탭, 폼의 `id`는 바꾸지 않습니다.

## 디자인만 수정할 때 권장 작업 범위

권장:

- `styles.css` 수정
- 필요한 경우 `index.html`의 문구/배치만 최소 수정

비권장:

- `app.js` 이벤트 함수 수정
- Supabase SQL 수정
- 버튼 id 변경
- form input name 변경
- data attribute 삭제
