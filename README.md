# 피부과 리뷰이벤트 MVP

배너 QR을 찍은 고객이 포토리뷰 완료 후 미니게임 상품뽑기 결과를 직원에게 보여주는 로컬 MVP입니다. 카카오톡 플친 추가는 시트팩 1장 추가증정 혜택으로 운영합니다.

## 실행

`index.html`을 브라우저로 열면 바로 사용할 수 있습니다.

## 포함된 흐름

1. 고객이 이름, 휴대폰 뒤 4자리, 네이버 ID 또는 리뷰 닉네임을 입력합니다.
2. 방문일은 시스템이 오늘 날짜로 자동 저장합니다.
3. 고객이 네이버 포토리뷰 작성 완료를 체크합니다.
4. 선택사항으로 카카오톡 플친 추가 혜택을 확인합니다.
5. 고객이 공개된 상품 확률을 확인하고 물방울 미니게임으로 상품뽑기를 1회 실행합니다.
6. 첫 결과가 마음에 들지 않으면 선택사항으로 설문조사를 완료하고 1회 재뽑기합니다.
7. 고객이 결과 화면과 네이버 리뷰 화면을 함께 직원에게 보여줍니다.
8. 직원이 확인 후 증정 완료 처리하고 차트 메모를 복사합니다.

## 어뷰징 방어 모델

- 같은 네이버 ID 또는 리뷰 닉네임은 마지막 참여일 기준 28일 이후 다시 참여할 수 있습니다.
- 같은 날 동일 네이버 ID 또는 같은 휴대폰/기기 조합은 기존 참여 내역으로 연결합니다.
- 포토리뷰 완료 체크 전 뽑기를 차단합니다.
- 카카오톡 플친 추가는 선택사항이며 완료 고객에게 시트팩 1장을 추가 증정합니다.
- 설문조사 완료는 선택사항이며 첫 결과가 마음에 들지 않을 때 1회 재뽑기 기회를 제공합니다.
- 재뽑기를 실행하면 기존 결과는 취소되고 새 결과 1개만 최종 지급 대상이 됩니다.
- 이미 당첨된 고객은 새로고침하거나 다시 클릭해도 기존 결과만 유지합니다.
- 뽑기 클릭 연타를 쿨다운과 진행 잠금으로 방어합니다.
- 재고가 0개인 상품은 자동으로 후보에서 제외합니다.
- 관리자는 상품명, 상품 설명, 재고, 확률 가중치를 수정하거나 새 상품을 추가할 수 있습니다.
- 고객에게는 상품명, 상품 설명, 공개 확률만 표시하며 재고 수량은 노출하지 않습니다.
- 네이버 ID 또는 리뷰 닉네임은 4주 참여 제한과 직원 확인을 위한 최소 식별값으로 사용합니다.
- 동일 이름과 휴대폰 뒤 4자리 재등장은 확인 필요 플래그로 표시합니다.
- 같은 기기에서 짧은 시간 내 반복 뽑기가 감지되면 확인 필요 플래그를 남깁니다.
- 등록, 리뷰 완료 체크, 카톡 추가혜택 확인, 설문 완료, 기본 뽑기, 재뽑기, 지급 완료는 감사 로그에 남습니다.
- 감사 로그는 단순 해시 체인으로 훼손 여부를 확인할 수 있습니다.

## 실제 배포 시 필수 변경

현재 MVP는 브라우저 `localStorage`에 저장되므로 운영용 보안 수단은 아닙니다. 실제 배포에서는 아래 로직을 서버 API로 이동해야 합니다.

- 배너에 삽입할 실제 QR URL
- 네이버 리뷰 작성 링크
- 설문조사 링크
- 참여자 고유키 생성과 중복 검사
- 카카오톡 채널 관계 확인
- 직원 권한 인증
- 상품뽑기 난수 생성과 재고 차감
- 당첨 결과 불변 저장
- 감사 로그 저장

권장 구조는 `고객 웹앱 + 관리자 콘솔 + 서버 API + DB + 파일 스토리지`입니다. 뽑기 결과는 클라이언트가 아니라 서버 트랜잭션에서 확정해야 합니다.
